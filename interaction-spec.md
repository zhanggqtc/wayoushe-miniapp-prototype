# 高校瓦友社小程序交互说明书

## 1. 项目定位

本小程序面向无畏契约高校用户，服务三类人群：

1. 已认证高校特权的瓦友社用户。
2. 未认证高校特权的潜在用户。
3. 高校先锋，协助官方组织活动和比赛。

产品目标不是做泛资讯社区，而是围绕高校身份、特权权益、活动参与、轻社交和积分激励，形成一个高校玩家每天愿意打开的运营阵地。

---

## 2. 底部导航

建议底部导航保持 4 个 Tab：

| Tab | 定位 | 核心内容 |
| --- | --- | --- |
| 瓦友社 | 首页，也是“我的高校瓦友社今日面板” | 高校身份、皮肤自选、活动雷达、积分入口、同校动态 |
| 活动 | 活动与比赛中心 | 官方活动、本校活动、附近网吧活动、比赛、先锋创建和管理入口 |
| 同好 | 高校同好雷达 | 每日推荐同校同好、轻互动、双向名片交换 |
| 我的 | 个人资产和记录 | 高校认证、皮肤权益、积分、活动记录、社交记录、先锋工具箱 |

高校先锋用户数量较少，不建议将“先锋工具”作为底部 Tab，而是通过身份解锁出现在活动页右上角和我的页工具箱中。

---

## 3. 首页：瓦友社

首页定位为“高校玩家每日控制台”。核心吸引力来自四件事：

1. 今天的高校特权皮肤随机。
2. 下月皮肤权益进度。
3. 本校或附近正在发生的活动。
4. 同校瓦友的存在感和轻社交入口。

### 3.1 高校身份卡

首页顶部展示当前用户所属高校瓦友社。

展示信息：

- 学校名称，例如“深圳大学瓦友社”。
- 高校特权认证状态。
- 瓦友社人数。
- 瓦友社等级。
- 本校先锋信息。

瓦友社等级包括：

| 等级 | 建议视觉 |
| --- | --- |
| 白银 | 银灰徽章 |
| 黄金 | 金色徽章 |
| 钻石 | 蓝紫或冷蓝徽章 |
| 神话 | 红黑电竞徽章 |
| 荣誉 | 金红特殊徽章 |

本校先锋状态：

- 有先锋：展示头像、昵称、身份说明，例如“负责活动与比赛组织”。
- 暂无先锋：展示“暂无先锋”，并提供弱入口“了解高校先锋计划”。

### 3.2 高校特权皮肤随机

该模块是首页最核心的日活钩子。

规则：

1. 每天每位玩家随机展示 3 款皮肤。
2. 当天结果固定，不可重抽。
3. 玩家可以将皮肤加入下月自选槽。
4. 自选槽最多 3 个。
5. 加入槽位不等于获得，只是“已占位”。
6. 完成任务后，该皮肤才会“已锁定”。
7. 月底结算时，未锁定或空槽由默认月度轮换皮肤补位。

核心术语：

| 术语 | 含义 |
| --- | --- |
| 今日可选 | 今日随机出的候选皮肤 |
| 加入自选槽 | 用户选择该皮肤进入下月候选 |
| 已占位 | 皮肤在槽位中，但任务未完成 |
| 待锁定 | 需要继续完成任务 |
| 已锁定 | 任务完成，下月确定生效 |
| 默认补位 | 空槽或未完成槽位由默认轮换皮肤补齐 |

### 3.3 自选槽状态

下月自选槽建议用 3 张状态差异明显的槽位卡展示。

#### 未选择

文案示例：

```text
槽位 3
未选择皮肤
月底将由默认轮换皮肤补位
```

视觉：暗色虚线框、空槽感、弱化按钮。

#### 已占位，待锁定

文案示例：

```text
槽位 1
盖亚复仇
状态：已占位，待锁定
任务：本周完成 5 场对局
进度：3/5
还差 2 场，完成后下月生效
```

视觉：橙色或黄色风险态，显示进度条或进度环，强调“待锁定”。

#### 已锁定

文案示例：

```text
槽位 2
离子幻象
状态：已锁定
任务已完成
下月高校特权生效
```

视觉：完成态，更稳定，可使用绿色状态标签和更强卡片质感。

### 3.4 积分入口

首页展示轻量积分资产卡。

文案示例：

```text
我的积分：1280
参加活动和组织比赛可获得积分
去商城兑换
```

注意：积分不来自每日登录、皮肤任务、同好互动或普通浏览。

### 3.5 活动雷达

首页展示少量强相关活动：

- 本校活动。
- 附近网吧活动。
- 官方活动。
- 即将截止报名。
- 今日可签到。
- 比赛即将开始。

活动雷达只放摘要，完整决策信息在活动 Tab。

### 3.6 同校动态

用于制造高校归属感，不做重社区信息流。

可展示：

- 今日活跃瓦友数。
- 正在冲自选皮肤人数。
- 开放报名活动数。
- 今日同好推荐数。

## 4. 活动 Tab

活动页定位为“我能参加什么、能获得什么、现在是否要报名”。

### 4.1 页面结构

顶部：

- 活动标题。
- 搜索或筛选入口。
- 高校先锋用户展示“创建”按钮。

筛选建议：

- 全部。
- 本校。
- 官方。
- 附近。
- 我的报名。
- 待签到。

### 4.2 活动卡信息

活动卡要比普通列表更丰富，建议做成“活动决策卡”。

基础信息：

- 活动名称。
- 活动类型：官方 / 本校 / 网吧 / 比赛。
- 活动状态：报名中 / 待审核 / 即将开始 / 进行中 / 已结束。
- 时间。
- 地点或线上线下。
- 距离，如果是附近网吧活动。
- 主办方：官方 / 本校先锋 / 网吧合作方。

参与信息：

- 报名截止时间。
- 已报名人数 / 名额上限。
- 是否需要组队。
- 是否需要高校特权认证。
- 签到方式。
- 比赛赛制。
- 当前是否可报名。

激励信息：

- 签到积分。
- 完赛积分。
- 名次奖励积分。
- 实物奖励。
- 是否计入瓦友社成长值。

普通玩家卡片示例：

```text
[本校比赛] 报名中
深大周末校内对抗赛
6 月 28 日 19:00 · 深圳大学活动中心
五人组队 · 已报名 32/64
报名截止：今晚 22:00
完赛 +100 积分
冠军额外 +500 积分
[报名]
```

网吧活动示例：

```text
[附近网吧] 预热中
南山网吧高校夜赛
距你 2.4km · 7 月 3 日 20:00
线下签到 · 限 48 人
签到 +40 积分
完赛 +80 积分
[查看详情]
```

官方活动示例：

```text
[官方活动] 进行中
高校夏日挑战赛
全高校开放 · 本周赛程更新
官方发起 · 无需高校先锋审核
[参与活动]
```

高校先锋视角示例：

```text
[待审核] 华南片区高校挑战赛
提交时间：今天 14:32
活动类型：核心比赛
预计报名：64 人
奖励：冠军 500 积分
需要官方审核后发布
[查看审核状态]
```

### 4.3 高校先锋活动工具箱

高校先锋通过活动页的“创建”进入工具箱。

第一版能力：

- 基于官方模板创建活动。
- 基于官方模板创建比赛。
- 填写可编辑内容框，例如学校、时间、地点、名额、奖励、注意事项。
- 提交官方审核。
- 管理报名。
- 管理队伍。
- 配置赛程与对阵。
- 录入结果。
- 生成比赛宣传海报。

审核规则：

- 官方发起活动免审核。
- 高校先锋发起的核心活动、比赛、含奖励活动必须审核。
- 普通活动可配置为轻审核或免审，但第一版建议偏审核。

组织积分：

- 前期只给高校先锋。
- 活动必须审核通过并真实开展后结算。
- 按实际签到、完赛人数、活动质量和官方协办程度发放。
- 不建议创建成功即发放积分。

## 5. 积分与商城

积分定位为“高校活动贡献积分”，具有现金等价成本，因此只投放在真实活动里。

### 5.1 积分来源

给积分：

- 活动签到。
- 活动完成。
- 比赛完赛。
- 比赛名次。
- 高校先锋成功组织活动或比赛。

不给积分：

- 每日登录。
- 皮肤任务。
- 同好互动。
- 普通浏览。
- 分享拉人。
- 报名但未参与。

### 5.2 商城边界

商城需要做在小程序内，但兑换之后的履约流程跳转外部成熟商城系统。

小程序内负责：

- 积分余额。
- 商品列表。
- 商品详情。
- 商品分类。
- 库存和限兑展示。
- 兑换确认。
- 兑换记录入口。
- 兑换后跳转。

外部系统负责：

- 收货地址。
- 订单履约。
- 发货物流。
- 售后。
- 最终库存一致性。

### 5.3 商城入口

不建议将商城作为底部 Tab。

入口包括：

1. 首页积分卡。
2. 我的页积分卡。
3. 活动完成后的获得积分结果页。
4. 活动详情页的积分奖励区域。

### 5.4 商城页面建议

商城首页：

```text
积分商城
我的积分：1280
兑换记录

推荐兑换
限定周边
校园活动物料
赛事奖励专区
```

商品卡：

- 商品图。
- 商品名称。
- 所需积分。
- 剩余库存。
- 每人限兑。
- 兑换按钮。

商品详情：

- 商品大图。
- 商品名称。
- 积分价格。
- 库存。
- 限兑规则。
- 兑换说明。
- 兑换后将跳转外部商城完成履约。

## 6. 同好雷达

同好模块定位为“发现同校里可能聊得来、玩得来的人”，不是强组队工具。

### 6.1 推荐机制

- 每天推荐 3 位同校同好。
- 画像主要来自游戏数据自动生成。
- 不要求用户每天答题。
- 不强制组队。

画像维度：

- 常用特工。
- 常玩位置或职责。
- 段位区间。
- 活跃时间段。
- 游戏风格。
- 参与活动或比赛倾向。

### 6.2 隐私规则

采用中隐私模式：

- 默认展示头像昵称。
- 展示系统生成标签。
- 展示段位区间。
- 展示同校身份。
- 游戏 ID、区服、联系方式需双方确认后可见。
- 不做开放私信广场。

### 6.3 互动方式

第一版采用克制混合模式：

- 点赞。
- 收藏。
- 预设打招呼。
- 双向感兴趣后交换游戏名片。
- 引导加入本校瓦友群。

## 7. 我的页

我的页定位为个人资产和记录中心。

内容：

- 高校特权认证状态。
- 所属高校。
- 游戏账号关联状态，复用已有高校特权认证体系。
- 是否高校先锋。
- 本月默认轮换皮肤。
- 下月自选皮肤槽位。
- 皮肤任务进度。
- 积分余额。
- 积分明细。
- 积分商城入口。
- 我的报名。
- 我的签到。
- 我的比赛队伍。
- 我的活动结果。
- 我感兴趣的人。
- 对我感兴趣的人。
- 已交换名片。
- 高校先锋工具箱。

## 8. AI 比赛海报

AI 海报第一版只服务比赛宣传，不扩展到社群日常运营。

入口：

- 比赛创建完成后。
- 先锋工具箱中的比赛管理页。

前端负责：

- 自动带入比赛名称、学校、时间、报名截止、赛制、奖励、报名入口。
- 选择海报风格。
- 发起生成。
- 展示 1-3 张候选海报。
- 保存、分享、设为比赛封面。

前端不负责：

- 模型选择。
- 素材合规。
- 敏感词审核。
- 图片任务队列。
- 图片存储。
- 官方审核规则。

## 9. 设计团队重点关注

### 9.1 首页视觉优先级

最高优先级：高校身份卡、今日皮肤候选、下月自选槽。

次级优先级：积分资产、活动雷达、同校动态。

### 9.2 皮肤状态差异

设计上必须拉开三种状态差异：

- 已占位，待锁定。
- 已锁定，下月生效。
- 未选择，默认补位。

避免让用户误解“选择了皮肤就已经获得”。

### 9.3 活动卡信息密度

活动卡需要承载决策信息，但不能变成后台表格。建议做清晰分区：

1. 类型和状态。
2. 活动名称和时间地点。
3. 报名和参与条件。
4. 积分和奖励。
5. 操作按钮。

### 9.4 积分商城

商城需要有前台展示体验，但不需要承接完整履约。设计上要明确“兑换后跳转外部系统完成后续流程”。

### 9.5 先锋能力

先锋能力应体现价值，但不要喧宾夺主。普通用户可看到“本校先锋”和“申请成为先锋”的弱入口，高校先锋身份下才展示创建、审核、海报、管理等能力。
